How the Platform Works

The three tools connect into a single workflow from crisis detection to funded response.

Step 1

Classify the Emergency

When a new crisis emerges, the Classification tool scores its severity using affected population data and country risk profiles. It determines whether the emergency warrants a White, Yellow, Orange, or Red response stance.

Step 2

Calculate CRF Allocation

For Orange and Red emergencies, the CRF Calculator auto-populates from the classification and calculates how much the Crisis Response Fund should allocate. Teams can compare coverage tiers and export funding summaries.

Step 3

Navigate the Response

The Response Navigator provides step-by-step guidance through IRC's emergency response protocols, helping teams track tasks, access resources, and coordinate their operational response.

How It's Built

This is a proof of concept built to demonstrate what's possible with modern web technologies and AI-assisted development.

Architecture

The platform is a monorepo with a shared package (@irc/shared) that provides common data, utilities, and UI components to all three apps. Each app deploys independently to GitHub Pages.

Frontend

Classification and CRF Calculator are vanilla HTML/CSS/JS with zero framework dependencies. The Navigator uses React with Vite. All apps share design tokens and the IRC brand system.

Backend & Database

Classification data is stored in Supabase (PostgreSQL) with row-level security. The shared package includes a REST client for fetching classifications. AI features use OpenRouter for model routing.

Development

Built collaboratively with AI-assisted development using Claude Code. CI/CD runs through GitHub Actions, building the shared package and deploying each app to its standalone repository's gh-pages branch.
